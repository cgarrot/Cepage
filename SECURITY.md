# Security Policy

## Supported versions

Cepage is currently in an early public phase.

Security fixes are handled on a best-effort basis for:

- the latest default branch
- the latest published release, if releases exist

## Reporting a vulnerability

Please do not open a public GitHub issue for a vulnerability report.

If this repository is hosted on GitHub with Security Advisories enabled, use the
private advisory flow. Otherwise, contact the maintainers privately through the
contact channel listed in the repository profile before sharing details
publicly.

When reporting an issue, include:

- a short description of the problem
- the affected component or file path
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation if you have one

We will acknowledge new reports as quickly as possible and aim to provide an
initial triage response within 5 business days.

## Scope

This policy covers:

- source code in this repository
- development tooling and scripts shipped with this repository
- documented local deployment paths

This policy does not cover:

- local environment mistakes outside the repository
- third-party hosted services you run independently
- public feature requests or non-sensitive hardening suggestions

## Disclosure

Please give maintainers reasonable time to investigate and prepare a fix before
public disclosure.
